Karate School Management System

A code-first ASP.NET Core MVC + EF Core application for running a karate dojo: student/instructor/administrator accounts, class scheduling, attendance, belt promotions, and payments — built from a UML class diagram and ERD.

This page is a static project overview. GitHub Pages can't run the app itself (no server, no database) — the live app is deployed separately.

Tech stack

ASP.NET Core MVC (.NET 8) EF Core, Code-First SQLite / SQL Server Custom cookie auth (PBKDF2) xUnit GitHub Actions CI

Architecture highlights

Domain model

Abstract User base class with Student, Instructor, and Administrator subclasses mapped via EF Core table-per-type inheritance, matching the source ERD's shared-key design.

Design patterns

UserFactory centralizes account creation (Factory pattern); IRepository<T> / UserRepository centralize data access and enforce one-account-per-email (Repository pattern).

Interfaces

IAttendanceTrackable, IPayable, and IReportable express cross-cutting student/admin capabilities independent of the class hierarchy.

Custom authentication

Cookie-based auth built on ASP.NET Core's cookie middleware (not Identity), with PBKDF2 password hashing, account lockout, and sliding session expiration.

Business rules enforced

  • Duplicate accounts — each email may only be associated with one User.
  • Minors — students under 18 require a guardian/emergency contact on file.
  • Account lockout — 5 consecutive failed logins locks the account for 15 minutes.
  • Sessions — expire after 30 minutes of inactivity (sliding cookie expiration).
  • Attendance — cannot be recorded for a future date.
  • Scheduling — a class can't exceed capacity, and a room or instructor can't be double-booked in the same time slot.
  • Belt promotions — instructor recommendations require justification notes and administrator approval before becoming final.
  • Passwords — minimum complexity policy, hashed with PBKDF2, never stored or transmitted in plain text.

Run it locally

git clone https://github.com/Ibrihim1986/Karateschool-managementsystem.git
cd Karateschool-managementsystem
dotnet run --project src/KarateSchool.Web

The database is created and seeded automatically on first run (SQLite by default).
